# 스터디 과제 관리 프로그램 요구사항

> 콘솔 환경에서 과목, 과제, 학습 계획, 학습 기록, 과제 메모를 관리하는 MySQL 연동 프로그램

---

## 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로그램명 | 스터디 과제 관리 프로그램 |
| 실행 방식 | 콘솔 기반 메뉴 프로그램 |
| 데이터베이스 | MySQL |
| 데이터베이스명 | `study_manager` |
| 주요 특징 | 5개 테이블 사용, 메인 메뉴와 하위 메뉴 구조 적용 |

이 프로그램은 사용자가 학습 과목과 과제를 등록하고, 학습 계획 및 학습 기록을 함께 관리할 수 있도록 한다.
또한 과제별 메모를 남길 수 있으며, 전체 학습 현황을 요약 정보로 확인할 수 있다.

---

## 2. 핵심 요구사항

- 총 5개의 테이블을 사용한다.
- 메뉴는 메인 메뉴와 하위 메뉴로 나누어 구성한다.
- 메인 메뉴 선택 후 하위 메뉴를 선택하는 중첩 조건문 구조를 사용한다.
- 각 기능은 `Service`에서 처리하고, 데이터베이스 작업은 `DAO`에서 처리한다.
- 입력값은 필수값, 날짜 형식, 시간 형식, 숫자 입력 여부를 검증한다.
- 우선순위와 상태는 `VARCHAR` 타입으로 저장한다.
- 우선순위와 상태 값 제한은 데이터베이스가 아니라 Python 코드에서 검사한다.

## 3. 데이터베이스 설계

### 3.1 테이블 목록

| 번호 | 테이블명 | 역할 |
| --- | --- | --- |
| 1 | `subjects` | 과목 정보 관리 |
| 2 | `tasks` | 과제 정보 관리 |
| 3 | `study_plans` | 학습 계획 관리 |
| 4 | `study_logs` | 학습 기록 관리 |
| 5 | `task_memos` | 과제별 메모 관리 |

### 3.2 테이블 상세

> 우선순위와 상태처럼 정해진 값만 입력해야 하는 항목은 데이터베이스에 `VARCHAR`로 저장한다.
> 입력 가능한 값인지는 Python의 조건문으로 검사한다.

#### subjects

| 컬럼명 | 자료형 | 설명 |
| --- | --- | --- |
| `subject_id` | `INT` | 과목 번호 |
| `subject_name` | `VARCHAR(50)` | 과목명 |
| `description` | `VARCHAR(200)` | 과목 설명 |
| `created_at` | `DATETIME` | 등록일 |

#### tasks

| 컬럼명 | 자료형 | 설명 |
| --- | --- | --- |
| `task_id` | `INT` | 과제 번호 |
| `subject_id` | `INT` | 과목 번호 |
| `title` | `VARCHAR(100)` | 과제 제목 |
| `content` | `TEXT` | 과제 내용 |
| `priority` | `VARCHAR(10)` | 우선순위 |
| `status` | `VARCHAR(10)` | 진행 상태 |
| `due_date` | `DATE` | 마감일 |
| `created_at` | `DATETIME` | 등록일 |

#### study_plans

| 컬럼명 | 자료형 | 설명 |
| --- | --- | --- |
| `plan_id` | `INT` | 계획 번호 |
| `subject_id` | `INT` | 과목 번호 |
| `plan_title` | `VARCHAR(100)` | 계획 제목 |
| `plan_date` | `DATE` | 계획 날짜 |
| `start_time` | `TIME` | 시작 시간 |
| `end_time` | `TIME` | 종료 시간 |
| `memo` | `VARCHAR(200)` | 메모 |
| `created_at` | `DATETIME` | 등록일 |

#### study_logs

| 컬럼명 | 자료형 | 설명 |
| --- | --- | --- |
| `log_id` | `INT` | 기록 번호 |
| `subject_id` | `INT` | 과목 번호 |
| `study_date` | `DATE` | 학습 날짜 |
| `study_time` | `INT` | 학습 시간 |
| `content` | `VARCHAR(300)` | 학습 내용 |
| `created_at` | `DATETIME` | 등록일 |

#### task_memos

| 컬럼명 | 자료형 | 설명 |
| --- | --- | --- |
| `memo_id` | `INT` | 메모 번호 |
| `task_id` | `INT` | 과제 번호 |
| `memo` | `VARCHAR(300)` | 메모 내용 |
| `created_at` | `DATETIME` | 등록일 |

## 4. 메뉴 구조

### 4.1 메인 메뉴

```text
===== Study Task Manager =====
1. 과목 관리
2. 과제 관리
3. 학습 계획 관리
4. 학습 기록 관리
5. 과제 메모 관리
6. 요약 정보
0. 종료
```

### 4.2 중첩 메뉴 처리 구조

메뉴는 다음과 같이 메인 메뉴와 하위 메뉴가 연결된 형태로 구현한다.

```python
if menu == 1:
    if sub_menu == 1:
        self.service.add_subject()
    elif sub_menu == 2:
        self.service.show_subjects()
    elif sub_menu == 3:
        self.service.delete_subject()

elif menu == 2:
    if sub_menu == 1:
        self.service.add_task()
    elif sub_menu == 2:
        self.service.show_all_tasks()
```

## 5. 기능 요구사항

### 5.1 과목 관리

```text
1. 과목 등록
2. 과목 전체 출력
3. 과목 삭제
0. 이전 메뉴
```

| 기능 | 설명 |
| --- | --- |
| 과목 등록 | 과목명과 설명을 입력받아 과목을 등록한다. |
| 과목 전체 출력 | 등록된 모든 과목을 출력한다. |
| 과목 삭제 | 과목 번호를 입력받아 과목을 삭제한다. |
| 삭제 제한 | 연결된 과제, 학습 계획, 학습 기록이 있으면 삭제할 수 없다. |

### 5.2 과제 관리

```text
1. 과제 등록
2. 과제 전체 출력
3. 조건별 과제 조회
4. 과제 검색
5. 과제 수정
6. 과제 완료 처리
7. 과제 삭제
0. 이전 메뉴
```

| 기능 | 설명 |
| --- | --- |
| 과제 등록 | 과목을 선택한 뒤 과제 제목, 내용, 우선순위, 마감일을 입력한다. |
| 과제 전체 출력 | 등록된 모든 과제를 출력한다. |
| 조건별 과제 조회 | 과목, 우선순위, 상태 조건으로 과제를 조회한다. |
| 과제 검색 | 제목 또는 내용에 포함된 검색어로 과제를 찾는다. |
| 과제 수정 | 과제 제목, 내용, 우선순위, 마감일을 수정한다. |
| 과제 완료 처리 | 과제 상태를 `완료`로 변경한다. |
| 과제 삭제 | 과제 번호를 입력받아 과제를 삭제한다. |

### 5.3 학습 계획 관리

```text
1. 학습 계획 등록
2. 학습 계획 전체 출력
3. 학습 계획 삭제
0. 이전 메뉴
```

| 기능 | 설명 |
| --- | --- |
| 학습 계획 등록 | 과목, 계획 제목, 계획 날짜, 시작 시간, 종료 시간, 메모를 입력한다. |
| 학습 계획 전체 출력 | 등록된 모든 학습 계획을 출력한다. |
| 학습 계획 삭제 | 계획 번호를 입력받아 학습 계획을 삭제한다. |

### 5.4 학습 기록 관리

```text
1. 학습 기록 등록
2. 학습 기록 전체 출력
3. 학습 기록 삭제
0. 이전 메뉴
```

| 기능 | 설명 |
| --- | --- |
| 학습 기록 등록 | 과목, 학습 날짜, 학습 시간, 학습 내용을 입력한다. |
| 학습 기록 전체 출력 | 등록된 모든 학습 기록을 출력한다. |
| 학습 기록 삭제 | 기록 번호를 입력받아 학습 기록을 삭제한다. |

### 5.5 과제 메모 관리

```text
1. 과제 메모 등록
2. 과제 메모 조회
3. 과제 메모 삭제
0. 이전 메뉴
```

| 기능 | 설명 |
| --- | --- |
| 과제 메모 등록 | 과제를 선택한 뒤 메모 내용을 등록한다. |
| 과제 메모 조회 | 전체 메모 또는 특정 과제의 메모를 조회한다. |
| 과제 메모 삭제 | 메모 번호를 입력받아 과제 메모를 삭제한다. |

### 5.6 요약 정보

요약 정보 메뉴에서는 다음 항목을 출력한다.

| 출력 항목 | 설명 |
| --- | --- |
| 전체 과목 수 | 등록된 과목 개수 |
| 전체 과제 수 | 등록된 과제 개수 |
| 완료 과제 수 | 상태가 `완료`인 과제 개수 |
| 대기 과제 수 | 상태가 `대기`인 과제 개수 |
| 학습 계획 수 | 등록된 학습 계획 개수 |
| 학습 기록 수 | 등록된 학습 기록 개수 |
| 총 학습 시간 | 모든 학습 기록의 학습 시간 합계 |
| 과제 메모 수 | 등록된 과제 메모 개수 |

## 6. 입력 검증 요구사항

| 검증 항목 | 조건 |
| --- | --- |
| 과목명 | 비워둘 수 없다. |
| 과제 제목 | 비워둘 수 없다. |
| 계획 제목 | 비워둘 수 없다. |
| 학습 내용 | 비워둘 수 없다. |
| 과제 메모 | 비워둘 수 없다. |
| 우선순위 | `낮음`, `보통`, `높음` 중 하나만 입력할 수 있다. |
| 과제 상태 | `대기`, `완료` 중 하나만 사용할 수 있다. |
| 날짜 | `YYYY-MM-DD` 형식으로 입력해야 한다. |
| 시간 | `HH:MM` 형식으로 입력해야 한다. |
| 번호 입력 | 숫자로 입력해야 한다. |

## 7. 파일 구성

| 파일명 | 역할 |
| --- | --- |
| `main.py` | 프로그램 실행 시작점 |
| `menu.py` | 메인 메뉴와 하위 메뉴 처리 |
| `models.py` | 테이블별 클래스 정의 |
| `study_service.py` | 사용자 입력, 출력, 기능 흐름 처리 |

| `db.py` | 데이터베이스 연결 정보 |
| `schema.sql` | 데이터베이스 및 테이블 생성 SQL |
| `requirements.md` | 프로그램 요구사항 문서 |

## 8. 완료 기준

- 5개 테이블이 정상적으로 생성된다.
- 메인 메뉴와 하위 메뉴가 정상적으로 출력된다.
- 각 하위 메뉴에서 등록, 조회, 수정, 삭제 기능이 실행된다.
- 잘못된 입력값에 대해 오류 메시지를 출력한다.
- 요약 정보에서 전체 현황을 확인할 수 있다.
