# 카페 주문 시스템 업그레이드

카페의 메뉴판, 주문 입력, 주문 누적, 카테고리별 매출 계산을 처리하는 프로그램을 작성하시오.

이번 문제의 핵심은 단순히 상품 금액만 계산하는 것이 아니라, 메뉴의 카테고리 정보를 함께 활용하여 **음료 매출**, **디저트 매출**, **전체 매출**, **할인 금액**, **최종 결제 금액**을 계산하는 것이다.

## 시작 코드

메뉴 정보는 `"상품명": ("카테고리", 가격)` 형태로 저장한다.

```python
menus = {
    "아메리카노": ("음료", 3000),
    "라떼": ("음료", 4000),
    "레몬에이드": ("음료", 4200),
    "쿠키": ("디저트", 2000),
    "케이크": ("디저트", 5000),
    "마카롱": ("디저트", 2500)
}

orders = {}
```

`orders` 딕셔너리에는 사용자가 주문한 상품명과 누적 수량을 저장한다.

```python
orders = {
    "아메리카노": 2,
    "쿠키": 3
}
```

## 명령어

프로그램이 시작되면 사용자는 명령어를 입력할 수 있다.

| 명령어 | 기능 |
| --- | --- |
| `메뉴` | 전체 메뉴판을 출력한다. |
| `주문` | 주문 모드를 시작한다. |
| `주문 완료` | 프로그램을 종료하고 최종 주문 내역과 매출 정보를 출력한다. |

`메뉴`, `주문`, `주문 완료`가 아닌 값을 입력하면 `"존재하지 않는 명령어입니다."`를 출력하고 다시 명령어를 입력받는다.

## 요구사항

### 1. 메뉴판 출력

사용자가 `메뉴`를 입력하면 전체 메뉴판을 출력한다.

- 메뉴판 제목은 `"=== 메뉴판 ==="`으로 출력한다.
- 각 메뉴는 `[카테고리] 상품명: 가격원` 형식으로 출력한다.
- 메뉴 출력 순서는 `menus` 딕셔너리에 저장된 순서를 따른다.

### 2. 주문 모드

사용자가 `주문`을 입력하면 주문 모드로 들어간다.

- 주문 모드가 시작되면 `"주문 모드를 시작합니다."`를 출력한다.
- 주문 모드에서는 `"상품명 수량"` 형식으로 주문을 입력받는다.
- `"종료"`를 입력하면 주문 모드를 끝내고 다시 명령어 입력으로 돌아간다.
- 주문 모드가 종료되면 `"주문 모드를 종료합니다."`를 출력한다.
- 메뉴판에 없는 상품을 입력하면 `"없는 메뉴입니다."`를 출력하고 주문을 계속 받는다.
- 같은 상품을 다시 주문하면 기존 수량에 누적한다.
- 주문 모드를 여러 번 실행해도 이전 주문 내역은 유지된다.

### 3. 최종 주문 내역과 매출 출력

사용자가 `주문 완료`를 입력하면 프로그램을 종료한다.

프로그램을 종료하기 전에 다음 내용을 출력한다.

1. 최종 주문 내역
2. 음료 매출
3. 디저트 매출
4. 전체 매출
5. 할인 금액
6. 결제 금액

최종 주문 내역은 `orders` 딕셔너리에 저장된 순서대로 출력한다.

각 상품의 카테고리와 가격은 `menus` 딕셔너리에서 가져온다.

```python
category, price = menus[item]
```

상품별 금액은 다음과 같이 계산한다.

```python
item_total = price * quantity
```

카테고리가 `"음료"`이면 음료 매출에 더하고, 카테고리가 `"디저트"`이면 디저트 매출에 더한다.

```python
if category == "음료":
    drink_total += item_total
elif category == "디저트":
    dessert_total += item_total
```

전체 매출은 음료 매출과 디저트 매출을 더한 값이다.

```python
total_price = drink_total + dessert_total
```

전체 매출이 `30000`원 이상이면 전체 매출의 `10%`를 할인한다. 그렇지 않으면 할인 금액은 `0`원이다.

결제 금액은 다음과 같이 계산한다.

```python
final_price = total_price - discount
```

## 구현 힌트

- `menus.items()`를 사용하면 상품명과 `(카테고리, 가격)`을 함께 꺼낼 수 있다.
- 주문 수량을 누적할 때는 `orders.get(item, 0)`을 활용할 수 있다.
- 주문 모드는 전체 프로그램 반복문 안에 또 다른 `while` 문을 넣어 만들 수 있다.
- 최종 매출 계산에서는 `orders.items()`를 순회하며 상품별 금액을 누적한다.

## 사용해 볼 개념

`while`, 중첩 반복문, `break`, `continue`, `split()`, `int()`, 딕셔너리, 딕셔너리 안의 튜플, 튜플 언패킹, `get()`, `items()`, 조건문, 누적 합계, f-string
